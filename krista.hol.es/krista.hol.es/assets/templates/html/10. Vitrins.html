Для того чтобы привести наименования витрин для отчетов новой версии к единообразию, было принято решение наименование для витрин задавать по следующему принципу:

12345
5-7 симв4-5 симв 0-5 симвфикс.фикс.
  • 1. Наименование региона\схемы. Желательно использовать Алиас базы, исключения возможны при длинных и некорректных наименованиях (обязательное, для всех витрин конкретного региона значение должно быть одинаковое) пример: MosObl
  • 2. Признак закачки\пакета\блока (обязательное) пример: FNS01
  • 3. Дополнительный признак к закачке\пакета\блока (не обязательное, заполняется в том случае если в рамках 1 закачки\пакета\блока могут закачиваться данные разной периодичности и разных источников. Например отчетности об исполнении бюджета и формы по кредиторской задолженности) пример: KzDz
  • 4. Периодичность обновления данных в витрине (обязательное, фиксированное) одно из значений:
  • • YEAR (год);
  • • MONTH (месяц);
  • • QUARTER (квартал);
  • • OPER (оперативно, последний день по которому есть данные);
  • • DAY (ежедневно, календарь, для выбора дня из списка)
  • • WEEK (еженедельно)
  • • CHANGES (по мере изменений)
  • 5. Тип базы данных (обязательное, фиксированное) одно из значений:
  • • RDB (РБД Oracle, Microsoft)
  • • MDX(МБД Oracle, Microsoft)
  • • DBF(разновидность РДБ)
  • • XML(по xml файлам)

Пример наименования витрины: MosObl_FNS01_MONTH_MDX

Кроме того, список используемых витрин необходимо вести в соответствующей ветке в всс:

Документация\Постановка задач\10_Web Интерфейс\Отчетность ПК\<Свой регион>

В документе указывать: наименование витрины, описание, используемый блок, периодичность.
